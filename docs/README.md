# docs

## Purpose

The **docs** folder contains the official documentation for **Project Ariadne**.

Unlike the KnowledgeVault, which stores information managed by Ariadne, this folder documents the design, philosophy, standards, and operation of Ariadne itself.

Think of this folder as the project's technical manual.

## Typical contents

Examples include:

- Project vision
- Engineering standards
- KnowledgeVault specifications
- Architecture documentation
- Design decisions
- Development guidelines
- User and developer guides
- Installation documentation
- Release notes
- Future roadmap

## What belongs here?

Anything that explains **how Ariadne is designed, built, or maintained** belongs in this folder.

These documents provide context for contributors and users and serve as the long-term reference for the project's evolution.

## What does *not* belong here?

This folder should not contain personal knowledge or Wiki content.

Examples that belong elsewhere include:

- Research notes
- AI concepts
- Journal entries
- Sources
- People
- General knowledge articles

Those belong within the KnowledgeVault.

## Design Principle

Project documentation should remain separate from managed knowledge.

This distinction allows Ariadne to document itself while simultaneously managing the knowledge of its users.

---

**In short:**

If the KnowledgeVault is the library, **docs** is the manual explaining how the library was designed, built, and maintained.
