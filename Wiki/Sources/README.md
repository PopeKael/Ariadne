# Sources

## Purpose

The **Sources** folder contains the original reference material from which knowledge is derived.

Sources provide the evidence that supports Wiki articles and ensure that knowledge within the KnowledgeVault remains transparent, traceable, and verifiable.

Unlike Wiki articles, which summarise and connect information, source documents remain as close as possible to their original form.

## Typical contents

Examples include:

- Research papers
- Articles
- Books
- Web pages
- PDFs
- Technical documentation
- White papers
- Meeting transcripts
- Videos and transcripts
- Personal reference material

## Workflow

Source material enters the KnowledgeVault through the ingestion pipeline.

During processing, Ariadne may:

- Extract text and metadata.
- Generate summaries.
- Identify entities and concepts.
- Link related information.
- Create or update Wiki articles.
- Preserve references back to the original material.

The original source remains available for verification and future analysis.

## What does *not* belong here?

This folder should not contain:

- Wiki articles
- Personal journals
- Project documentation
- AI-generated knowledge summaries
- Processing artefacts

Sources are evidence, not conclusions.

## Design Principle

Knowledge should always be traceable to its origins.

By preserving original source material, Ariadne enables every piece of knowledge to be verified, revisited, and reinterpreted as understanding evolves.

---

**In short:**

The **Sources** folder is Ariadne's reference library, preserving the original evidence from which trusted knowledge is built.
