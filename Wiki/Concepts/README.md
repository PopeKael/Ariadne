# Concepts

## Purpose

The **Concepts** folder contains Wiki articles that explain ideas, principles, theories, and abstract topics.

Concepts provide the foundational knowledge that supports understanding across the KnowledgeVault. They describe *what something is*, *how it works*, and *why it matters*, independent of any particular project or implementation.

## Typical contents

Examples include:

- Artificial Intelligence
- Retrieval-Augmented Generation (RAG)
- Embeddings
- Vector Databases
- Knowledge Graphs
- Object-Oriented Programming
- Networking Concepts
- Cybersecurity Principles
- Machine Learning
- Prompt Engineering

## Workflow

Concept articles are created as knowledge is developed and verified.

They may:

- Reference supporting source material.
- Link to related concepts.
- Be referenced by Projects.
- Support technical documentation.
- Provide context for practical implementations.

Concepts evolve over time as understanding improves, making them living documents rather than static references.

## What does *not* belong here?

This folder should not contain:

- Project documentation
- Research notes
- Journal entries
- Raw source material
- System documentation
- Temporary working notes

These belong within their respective folders elsewhere in the KnowledgeVault.

## Design Principle

Concepts describe ideas that remain useful beyond any single project.

By separating conceptual knowledge from implementation details, Ariadne builds a reusable foundation that supports learning, reasoning, and future knowledge discovery.

---

**In short:**

The **Concepts** folder is Ariadne's textbook, explaining the ideas that underpin everything else in the KnowledgeVault.
