# People

## Purpose

The **People** folder contains information about individuals who have a meaningful relationship to the KnowledgeVault.

These records provide context about people referenced throughout the vault and enable Ariadne to maintain consistent identities, relationships, and historical context across multiple notes and projects.

This folder is intentionally excluded from the public repository, as its contents are expected to be personal to each user.

## Typical contents

Examples include:

- Personal profiles
- Family members
- Friends
- Colleagues
- Clients
- Contacts
- Historical relationships
- Biographical notes

## Workflow

People records are created and updated as Ariadne encounters individuals throughout the KnowledgeVault.

These records may be referenced from:

- Journal entries
- Projects
- Wiki articles
- Sources
- Entities

Maintaining a single record for each individual helps Ariadne avoid duplication while preserving context and relationships.

## Privacy

The contents of this folder are considered user-specific.

For this reason, only this README file is included in the public repository. Individual profile files are intentionally excluded from version control.

Each user's KnowledgeVault will naturally develop its own collection of people over time.

## What does *not* belong here?

This folder should not contain general research, project documentation, or Wiki articles.

Only information relating to identifiable individuals belongs here.

## Design Principle

People are often central to knowledge.

By maintaining dedicated records for individuals, Ariadne can build richer connections between ideas, conversations, projects, and historical events while respecting each user's private data.

---

**In short:**

The **People** folder is Ariadne's private address book, preserving the people behind the knowledge while keeping personal information outside the public project.
