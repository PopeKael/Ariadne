# Archive

## Purpose

The **Archive** folder contains information that has reached the end of its active lifecycle but is still worth preserving.

Archived material is retained for historical reference, auditing, backup, or future research, while keeping the active KnowledgeVault focused and uncluttered.

Think of this folder as Ariadne's long-term storage.

## Typical contents

Examples include:

- Superseded documents
- Previous versions of knowledge
- Retired projects
- Historical exports
- Completed processing batches
- Compressed snapshots of processed data
- Long-term backups

## When should information be archived?

Information should be moved here when it is:

- No longer actively maintained.
- No longer required for day-to-day retrieval.
- Retained for historical or audit purposes.
- Replaced by newer or more accurate knowledge.

## What does *not* belong here?

This folder should not be used as a temporary holding area or recycle bin.

Items that still require processing, review, or verification belong in their respective workflow folders until their lifecycle is complete.

## Design Principle

Ariadne values knowledge preservation without allowing historical information to overwhelm current knowledge.

Archiving keeps the active vault efficient while ensuring that previous work is never lost.

---

**In short:**

If the library shelves become crowded, the Archive is the climate-controlled basement where every book is carefully catalogued and can still be retrieved when needed.
