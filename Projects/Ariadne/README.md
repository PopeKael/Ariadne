# Project Ariadne

## Overview

Ariadne is the primary development project within the KnowledgeVault.

Its purpose is to build a trusted AI Librarian: a long-term personal intelligence system that preserves knowledge, understands context, records decisions, and intelligently routes work to the most appropriate AI model or tool.

Unlike a traditional chatbot, Ariadne is designed to become a persistent thinking companion whose value increases over time as its knowledge base grows.

---

## Purpose of this Folder

This directory contains the working documentation for the Ariadne project.

The files here describe how Ariadne is designed, why architectural decisions were made, how the project has evolved, and where development is heading.

This is the project's engineering workspace rather than the knowledge repository itself.

---

## Directory Structure

### Architecture

System architecture, diagrams, component layouts and design evolution.

### Chronicle

A chronological record of the project's evolution.

Each entry explains:

- what problem was being solved
- what was learned
- what decisions were made
- why those decisions were made

### Decisions

Architectural Decision Records (ADRs).

These documents explain important engineering choices and their reasoning so future development understands both the implementation and the intent.

### Releases

Milestone summaries for each development version.

Each release records:

- objectives
- completed work
- major changes
- known limitations
- next objectives

---

## Relationship to the KnowledgeVault

The KnowledgeVault is the permanent knowledge repository.

Ariadne is one project within that repository.

The KnowledgeVault stores knowledge.

Ariadne provides the architecture that organises, understands and works with that knowledge.

---

## Development Philosophy

- Knowledge belongs to the user.
- Markdown is the permanent storage format.
- AI models are interchangeable tools.
- Every significant decision should be documented.
- Documentation evolves alongside the software.
- The project should remain understandable years after it was built.
