# Logs

## Purpose

The **Log** folder contains operational records generated by Ariadne during normal operation.

These logs provide a chronological record of system activity, allowing processing to be monitored, audited, and troubleshot without affecting the KnowledgeVault itself.

Unlike the Journal, which records human observations and ideas, the Log records the actions performed by Ariadne.

## Typical contents

Examples include:

- Processing logs
- Import summaries
- Verification reports
- Error reports
- Pipeline execution history
- Batch processing results
- Performance statistics
- Diagnostic information

## Workflow

Log files are generated automatically by Ariadne as tasks are performed.

Depending on project configuration, logs may be:

- Rotated periodically.
- Archived after a defined period.
- Used for troubleshooting.
- Referenced during system validation.
- Deleted when no longer required.

## What does *not* belong here?

This folder should not contain personal notes, research, Wiki articles, or captured knowledge.

Logs exist solely to record the operational history of the system.

## Design Principle

Ariadne should always be able to explain **what it did, when it did it, and whether it succeeded**.

Maintaining a clear operational history improves transparency, troubleshooting, and confidence in the knowledge processing pipeline.

---

**In short:**

The **Journal** records the story of the user.

The **Log** records the story of the system.
