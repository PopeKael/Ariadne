Ariadne System Reference

A working index of 00_System: what runs the vault, what describes it, and which files are state or recovery artefacts. This is intentionally a reference page, not another control plane.

Vault-relative location: 00_System/System-Reference.html · Inventory reviewed 2026-07-14

Operating model

1. Ingest

ariadne.ps1 classifies Inbox Markdown, creates review/catalogue records, links it, then files it.

2. Connect

Graph scripts maintain explicit relationships, concept pages, entities, and person identity records.

3. Improve

The compiler proposes or applies conservative Wiki links. It is deliberately separate from ingest.

4. Retrieve

ariadne_mcp.py exposes the catalogue and Processed corpus read-only to MCP clients.

Practical rule: treat the red scripts as state-changing maintenance tools, not as routine housekeeping. library.json is authoritative catalogue state; make a Git commit or copy before graph-wide migration/rebuild work.

The backup files are a useful short-term safety net, but Git history is the cleaner long-term rollback mechanism. Multiple independent snapshots can otherwise become archaeological strata with no retention policy.

Recommended routine

Use Run Injest.ps1 (or ariadne.ps1) for normal intake. Review the generated Review files and run Compile-Knowledge.ps1 -Mode Proposal when you want to improve the Wiki graph intentionally. Use GraphHealth.ps1 to measure the result. Reserve migration, rebuild, and repair scripts for a diagnosed problem; they are scalpels, despite the temptation to use them as a leaf blower.
