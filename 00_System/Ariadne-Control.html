Ariadne Control

Launch the normal KnowledgeVault workflows here. Each action opens in its own PowerShell window, so you can see output and failures directly. This page works only while Start-AriadneControl.ps1 is running; press Ctrl+C in that window to stop it.

Operations

Process Inbox

Classify and file pending Inbox items, updating the catalogue and knowledge map.

Reclassify Entire Vault

Reprocess all Processed, Failed, and Inbox documents using the current taxonomy, then rebuild the graph and published views.

Reclassification Status

Show completed, remaining, percentage, current document, elapsed time, and failures for the full-vault run.

Retry Failed Ingestion

Move retryable failed files back to Inbox. Run ingest afterwards to process them.

Knowledge Link Proposals

Create conservative Wiki-link proposals without changing notes.

Graph Health Audit

Write the current graph-health report without modifying the graph.

Publish Knowledge Views

Regenerate the Wiki knowledge map and searchable index from canonical system data.

Embedding Index Status

Check whether the local semantic-search index is present and compatible.

Maintenance

These workflows change generated state or repository contents. The red actions are broad or migration-style operations; take a snapshot first.

Rebuild Embedding Index

Discard and regenerate the local embedding index. It remains local and is not committed to Git.

Reconcile Graph

Repair one-way links and add bounded semantic links for weakly connected notes.

Rebuild Graph Relationships

Recalculate bounded relationships across the complete catalogue.

Repair Retry Queue

Normalise and deduplicate the ingestion retry queue.

Migrate Legacy Graph

Run the legacy graph migration. Intended for an upgrade, not routine use.

Migrate Person Entities

Migrate older person entity data to the dedicated identity model.

Commit and Push Snapshot

Stage all current changes, create a timestamped commit, and push main.
