Ariadne Control

Launch supported rebuild-v1 workflows here. Each action opens in its own PowerShell window, so you can see output and failures directly. This page works only while Start-AriadneControl.ps1 is running; press Ctrl+C in that window to stop it.

Operations

Process Inbox

Run rebuild-v1 intake for genuinely new Inbox files: stable IDs, review-only enrichment, catalogue update, filing, and embeddings.

Embedding Index Status

Check whether the local semantic-search index is present and compatible.

Evaluate Retrieval

Run the versioned MCP chunk-retrieval benchmark. It does not alter vault data.

Rebuild Regression Tests

Run the rebuild-v1 Python test suite. This is developer validation, not daily ingestion.

Preview Downloads Organisation

Show what the Downloads organiser would file: Markdown to Inbox, emails to Docs, and media to their download folders. No files move.

Maintenance

These workflows change derived state or generate diagnostic reports. Failure remediation remains deliberately command-driven after a fresh audit; it is not a one-click menu action.

Rebuild Embedding Index

Discard and regenerate the local embedding index. It remains local and is not committed to Git.

Audit Failed Ingestion

Create a read-only rebuild-v1 failure report from retained capture and rejection evidence.

Organise Downloads

Move top-level Downloads files using the previewed rules. It skips filename collisions and does not overwrite files.
